Домашнее задание № 2.

Спроектируйте схему базы данных для социальной сети.

В проектируемой базе данных должна быть возможность хранить данные для социальной сети, в которой присутствуют:
  - пользователи, для которых необходимо хранить, как минимум: имя, дату рождения, номер телефона, e-mail, а также друзей и родственников(родители, дети, муж/жена, братья и сёстры);
  - личные сообщения пользователей;
  - стена пользователя, т.е. набор текстовых сообщений оставленных публично для пользователя другим пользователем или группой(см. ниже);
  - группы(сообщества), для которых необходимо хранить, как минимум: имя группы, создателя сообщества, администраторов, модераторов;
  - стена группы, т.е. набор текстовых сообщений оставленных публично для пользователя другим пользователем или группой;
  - комментарии к сообщениям на стенах сообществ и пользователей.
Описанная схема должна обрабатывать проверки целостности ссылок между таблицами.

Вам необходимо смоделировать схему БД и описать на языке DDL все элементы схемы .
Результат должен быть представлен в виде файла с расширением '.sql', содержащий команды на языке SQL для создания всех необходимых элементов базы данных.
Полученный файл должен корректно исполняться в выбранной вами СУБД в пакетном режиме.
Пример такого файла с некоторой простой БД можно найти в приложении к этому письму.
